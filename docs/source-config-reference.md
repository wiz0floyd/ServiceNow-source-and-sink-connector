# Source Connector Configuration Reference

Complete reference for all configuration properties accepted by `HermesSourceConnector`. Properties marked **Sensitive** are declared `Type.PASSWORD` in the connector's `ConfigDef` ‚Äî Kafka Connect masks them in REST responses and log output. Mark them as sensitive when uploading to Confluent Cloud so they are encrypted at rest.

For the canonical sensitive property list, see `HermesSourceConfig.SENSITIVE_PROPERTIES_CSV` in the connector source.

---

## Required properties

| Property | Type | Default | Description |
|----------|------|---------|-------------|
| `hermes.instance.name` | String | *(required)* | ServiceNow instance name. Use the bare subdomain only ‚Äî e.g., `myinstance` from `myinstance.service-now.com`. Must not include the protocol (`https://`), domain suffix (`.service-now.com`), or any path. The connector derives Hermes bootstrap endpoints for both peer clusters from this value. |
| `hermes.source.topic` | String | *(required)* | Full Hermes topic name to consume from. Example: `snc.myinstance.sn_streamconnect.orders`. The topic must exist on both Hermes peer clusters before the connector starts. |
| `hermes.consumer.group.id` | String | *(required)* | Kafka consumer group ID used by the embedded consumers connecting to both Hermes peer clusters. Hermes ACLs require the group ID to be prefixed `snc.<instanceName>.` ‚Äî the connector applies this prefix automatically if it is not already present in the value you supply. |
| `hermes.destination.topic` | String | *(required)* | Destination topic in Confluent Cloud to which records from Hermes are forwarded. |
| `hermes.ssl.keystore.b64` | Password | *(required when `hermes.ssl.enabled=true`)* | **Sensitive.** Base64-encoded PKCS12 keystore generated by the ServiceNow Instance PKI Certificate Generator. Encode with `openssl base64 -A -in keystore.p12` (the `-A` flag is mandatory ‚Äî line breaks in the string cause parsing failures). |
| `hermes.ssl.keystore.password` | Password | *(required when `hermes.ssl.enabled=true`)* | **Sensitive.** Password for the PKCS12 keystore in `hermes.ssl.keystore.b64`. |
| `hermes.ssl.truststore.b64` | Password | *(required when `hermes.ssl.enabled=true`)* | **Sensitive.** Base64-encoded PKCS12 truststore generated by the ServiceNow Instance PKI Certificate Generator. Encode with `openssl base64 -A -in truststore.p12`. |
| `hermes.ssl.truststore.password` | Password | *(required when `hermes.ssl.enabled=true`)* | **Sensitive.** Password for the PKCS12 truststore in `hermes.ssl.truststore.b64`. |

## Optional properties

| Property | Type | Default | Description |
|----------|------|---------|-------------|
| `hermes.consumer.max.poll.records` | Int | `500` | Maximum number of records returned in a single `poll()` call by each embedded Hermes consumer. The connector runs two consumers (one per peer cluster), so the maximum records buffered per poll cycle is `2 √ó hermes.consumer.max.poll.records`. Increase for higher throughput; decrease if task heartbeats are timing out. Must be ‚â• 1. |
| `hermes.ssl.enabled` | Boolean | `true` | Set to `false` to disable mTLS and use plaintext. **For local development and Docker E2E testing only ‚Äî never disable SSL in production.** When `false`, the four `hermes.ssl.*` properties are not required. |
| `hermes.source.cluster1.bootstrap.override` | String | *(empty ‚Äî derived automatically)* | Bootstrap servers override for Hermes peer cluster 1. When set, replaces the automatically derived `<instance>.service-now.com:4100-4107` address. For local development / Docker E2E testing only. Accepts the standard Kafka `bootstrap.servers` format, e.g., `localhost:19092`. |
| `hermes.source.cluster2.bootstrap.override` | String | *(empty ‚Äî derived automatically)* | Bootstrap servers override for Hermes peer cluster 2. When set, replaces the automatically derived `<instance>.service-now.com:4200-4207` address. For local development / Docker E2E testing only. |

## Standard Connect properties

These are required by the Kafka Connect framework and are not connector-specific:

| Property | Recommended value | Description |
|----------|------------------|-------------|
| `connector.class` | `com.servicenow.kafka.connect.hermes.HermesSourceConnector` | Connector Java class |
| `tasks.max` | `1` | The connector always runs exactly one task regardless of this setting. The effective maximum is 1 per connector instance. For higher throughput, deploy additional connector instances. |

## Consumer group ID prefix behavior

Hermes enforces that consumer group IDs start with `snc.<instanceName>.` via broker ACLs. If a consumer connects with a non-prefixed group ID, Hermes returns `GroupAuthorizationException`.

The connector normalizes the group ID at startup:

- If `hermes.consumer.group.id` already starts with `snc.<instanceName>.`, it is used as-is.
- Otherwise, the connector prepends `snc.<instanceName>.` before submitting to the broker.

This means `hermes-connect-source` and `snc.myinstance.hermes-connect-source` are equivalent when `hermes.instance.name=myinstance`.

## Sensitive properties summary

The following properties must be declared as sensitive when uploading the connector plugin to Confluent Cloud:

```
hermes.ssl.keystore.b64,hermes.ssl.keystore.password,hermes.ssl.truststore.b64,hermes.ssl.truststore.password
```

Pass this CSV string to the `--sensitive-properties` flag when using the CLI:

```bash
confluent connect custom-plugin create hermes-source-connector \
  ...
  --sensitive-properties hermes.ssl.keystore.b64,hermes.ssl.keystore.password,hermes.ssl.truststore.b64,hermes.ssl.truststore.password
```

## Bootstrap endpoint derivation

When neither `hermes.source.cluster1.bootstrap.override` nor `hermes.source.cluster2.bootstrap.override` is set, the connector derives the bootstrap addresses from `hermes.instance.name`:

| Cluster | Derived bootstrap |
|---------|------------------|
| Peer cluster 1 | `<instance>.service-now.com:4100,<instance>.service-now.com:4101,...,<instance>.service-now.com:4107` |
| Peer cluster 2 | `<instance>.service-now.com:4200,<instance>.service-now.com:4201,...,<instance>.service-now.com:4207` |

The first 8 ports of each range (4100‚Äì4107 and 4200‚Äì4207) are used as bootstrap servers; the full port range 4100‚Äì4150 and 4200‚Äì4250 represents the broker pool reachable after bootstrap.
