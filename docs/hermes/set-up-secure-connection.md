# Set up a secure connection to the Hermes Messaging Service

Source: https://www.servicenow.com/docs/r/servicenow-platform/multi-instance-framework-hermes/set-up-secure-connection-to-hermes.html
Retrieved: 2026-05-20

## Procedure

1. Navigate to All > Certificate Generator > Instance PKI Certificate Generator.
2. Optionally configure ACLs (namespace-level or topic-level, Read Only or Read/Write).
3. Enter a keystore password in the Certificate Password field. Select Generate.
   - The system generates an instance-signed certificate, creates a keystore, and creates a truststore.
4. Select **Download Keystore** — save the PKCS12 keystore file.
5. Select **Download Truststore** — save the PKCS12 truststore file.
6. Copy both files to each producer and consumer client that will connect to Hermes.

## Notes

- You must use the keystore generated by Instance PKI Certificate Generator. Custom-generated keystores are not supported.
- Roles required: hermes_admin, sn_kmf.cryptographic_manager, or admin.

## SSL properties (producer.properties / consumer.properties)

```
security.protocol=SSL
ssl.truststore.password=<truststore password>
ssl.truststore.location=<path to truststore.p12>
ssl.truststore.type=PKCS12
ssl.keystore.password=<keystore password>
ssl.keystore.location=<path to keystore.p12>
ssl.keystore.type=PKCS12
ssl.key.password=<keystore password>
```
