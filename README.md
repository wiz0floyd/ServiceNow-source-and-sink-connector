## What it does

- Reads from a Confluent Cloud topic (any data â€” pass-through bytes).
- Writes to a configured Hermes topic on your ServiceNow instance.
- Authenticates via mTLS using a certificate generated by the ServiceNow Instance PKI Certificate Generator.
- Preserves all Kafka record headers verbatim.
- Fails loud at startup if the Hermes topic does not exist (no auto-create).

## Why it exists

Hermes is the Kafka cluster that fronts ServiceNow's Stream Connect data plane. Confluent's Cluster Linking does not solve the Hermes-as-destination direction. This connector ships as a Confluent Cloud Custom Connector so customers can move data from Confluent Cloud into ServiceNow workflows without VPN, PrivateLink, or a self-hosted Connect cluster.

## Requirements

- ServiceNow instance with the Hermes Messaging Service activated
- An mTLS certificate generated via **Instance PKI Certificate Generator** (PKCS12 keystore and truststore)
- Confluent Cloud cluster on a public-internet or PrivateLink-enabled endpoint

## Source connector

The package also includes a source connector (`HermesSourceConnector`) that reads from a Hermes topic and writes to a Confluent Cloud topic. It runs two embedded `KafkaConsumer` instances â€” one per Hermes peer cluster (ports 4100â€“4150 and 4200â€“4250) â€” inside a single task.

`tasks.max` is effectively **1 per connector instance**. The dual-cluster state lives inside that one task and cannot be split across tasks. For higher throughput, deploy additional connector instances; each instance manages its own independent dual-consumer pair.

See the [Quick Start â€” Source](docs/quick-start-source.md), [Source Config Reference](docs/source-config-reference.md), and the sample [`config/quickstart-hermes-source.properties`](config/quickstart-hermes-source.properties).

## Delivery semantics

End-to-end **at-least-once**. The producer is configured with `enable.idempotence=true` and `acks=all`. Idempotency / dedup is the consumer's responsibility on the Hermes side.

## Sensitive properties

All four mTLS-related configs are declared `Type.PASSWORD` in the connector's `ConfigDef`, which causes Kafka Connect to mask them in REST and log output. When uploading to Confluent Cloud, also pass them via the platform-level sensitive list so they are encrypted at rest:

```
hermes.ssl.keystore.b64,hermes.ssl.keystore.password,hermes.ssl.truststore.b64,hermes.ssl.truststore.password
```

## Performance

### Sink Connector
The sink connector uses an async send pipeline: records are dispatched to the Hermes producer without blocking, and offsets are committed only after `flush()` confirms delivery. Throughput scales with Hermes broker capacity and network RTT.

**Key tuning levers:**
| Config | Default | Effect |
|--------|---------|--------|
| `hermes.producer.linger.ms` | 5 | Increase to coalesce small batches (improves throughput at cost of latency) |
| `hermes.producer.batch.size` | 16384 | Max bytes per batch |
| `hermes.producer.compression.type` | none | `lz4` recommended for CPU/throughput trade-off |
| `tasks.max` | 1 | Each task opens one producer; increase for parallel sink tasks |

### Source Connector
The source connector runs two embedded KafkaConsumers (one per Hermes peer cluster) within a single task. The effective throughput limit is `hermes.consumer.max.poll.records Ă— 2` records per poll cycle.

**Key tuning levers:**
| Config | Default | Effect |
|--------|---------|--------|
| `hermes.consumer.max.poll.records` | 500 | Records per consumer per poll cycle (max 1000 total/cycle) |
| `hermes.consumer.poll.timeout.ms` | 100 | Max ms to block per cluster; lower = lower latency, higher CPU |
| Deploy additional instances | â€” | Each instance adds one task with two consumers |

## Security Model

### mTLS authentication

All connections to Hermes use mutual TLS (mTLS). The connector presents a PKCS12 client certificate generated by the ServiceNow Instance PKI Certificate Generator. The server certificate is validated against a truststore from the same generator. There is no fallback to SASL/SCRAM or OAUTHBEARER â€” the Hermes broker rejects any non-mTLS connection.

### Certificate storage

The keystore and truststore are loaded from base64-encoded strings in the connector configuration, not from file paths. This is required because Confluent Cloud Custom Connectors inject secrets as config values, not files. The raw bytes are held in a `KeyStore` in heap memory inside `InMemorySslEngineFactory` (KIP-519). They are declared `Type.PASSWORD` in the `ConfigDef` so Kafka Connect masks them in REST responses and logs. Confluent Cloud additionally encrypts them at rest when the properties are listed in `--sensitive-properties`.

### Hostname verification

`InMemorySslEngineFactory` sets `endpointIdentificationAlgorithm` to an empty string (disabled). This is intentional and not a security defect. The Hermes broker presents a certificate signed by the ServiceNow instance PKI â€” the CN matches the instance's internal host identity, which does not match the bootstrap address format that Connect clients use externally. Disabling JSSE hostname verification is safe here because:

1. **Bootstrap address is derived from the instance name you provided.** The connector constructs bootstrap addresses from `<instanceName>.service-now.com` + known port ranges. An attacker would have to intercept that specific DNS name to present a certificate for a different instance.
2. **The server certificate is validated against the instance truststore.** The truststore contains only the CA certificate for your ServiceNow instance's PKI. A certificate from any other CA â€” including well-known public CAs â€” is rejected. This is equivalent to certificate pinning.
3. **mTLS is mutual.** The broker also validates the client certificate. A man-in-the-middle cannot relay a valid client cert to a different Hermes instance.

If your threat model requires hostname verification in addition to these controls, you can extend `InMemorySslEngineFactory.createClientSslEngine()` to call `engine.setSSLParameters()` with an `SSLParameters` object that sets `endpointIdentificationAlgorithm = "HTTPS"`.
